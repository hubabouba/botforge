# Roadmap — AI-лаборатория для создания ботов

> Продукт: веб-платформа, где ИИ пишет **настоящий рабочий код** Telegram/Discord
> ботов. Рабочее место — как IDE: файловое дерево, редактор кода, ИИ-чат с ходом
> рассуждений, авто-исправление ошибок, вкладки **Код / Архитектура / Логи /
> Аналитика**, кнопки **Запустить / Скачать ZIP / Поделиться**. Ощущение —
> премиальная «AI-лаборатория», не шаблонный ИИ-сайт.

## Стек (финальный)
| Слой | Технология | Почему |
|------|-----------|--------|
| Фреймворк | **Next.js 15** (App Router, TS) | фронт+бэк на одном языке, деплой на Vercel |
| БД + Auth | **Supabase** (Postgres + Auth + RLS + Storage) | БД, авторизация и хранение — из коробки |
| Платежи | **Stripe** (Subscriptions: Basic/Pro, webhooks) | подписочная модель |
| Хостинг | **Vercel** (Preview + Production) | нативно для Next.js, превью на каждый PR |
| Мониторинг ошибок | **Sentry** (client + server + edge) | трейсы ошибок, source maps |
| Продуктовая аналитика | **PostHog** (события, воронки, session replay) | вкладка «Аналитика», продуктовые метрики |
| ИИ | **Anthropic Claude** (`claude-sonnet-5` ген, стриминг) | генерация/правка мульти-файлового кода |
| Редактор кода | **Monaco** (`@monaco-editor/react`) | тот же движок, что в VS Code |
| UI | **Tailwind + hand-crafted компоненты** (паттерны shadcn/ui) | уникальный вид, не «ИИ-шаблон» |
| CI/CD | **GitHub + GitHub Actions** | lint/typecheck/test/build на каждый PR |
| Валидация | **Zod** везде на границах | меньше багов |

**Готовые решения, которые переиспользуем (не изобретаем велосипед):**
Supabase Auth Helpers для Next.js, Stripe `subscriptions` sample-паттерн,
Monaco React, `@t3-oss/env-nextjs` для типобезопасных env, shadcn/ui-паттерны,
Sentry+PostHog официальные Next.js SDK.

## Модель данных (Supabase, с Row Level Security)
- `profiles` (1-к-1 с `auth.users`) — план, лимиты, дата
- `projects` — бот-проект пользователя (название, платформа, статус)
- `files` — файлы проекта (path, content) *(или JSONB-снимок в `projects`)*
- `messages` — история ИИ-чата проекта (роль, контент, reasoning)
- `runs` / `logs` — запуски и логи выполнения
- `subscriptions` — зеркало Stripe (customer_id, price_id, status, period_end)
- `events` — легкие продуктовые события (дублируем в PostHog)
RLS: пользователь видит только свои строки. Service-role — только на сервере.

## Фазы (делаем частями, каждая — самостоятельный рабочий кусок)
- **Фаза 0 — Фундамент/инфра**: репо, GitHub Actions (lint+typecheck+build),
  типобезопасные env, каркасы Sentry+PostHog, `.env.example` со всеми ключами,
  структура папок, базовый layout. *(деплой на Vercel готовим, включаем в Ф2)*
- **Фаза 1 — Дизайн-система + Лендинг** ← **делаем первой**: токены (цвета,
  типографика, тени, радиусы), UI-примитивы (Button, Input, Card, Dialog,
  Tabs…), красивый премиальный лендинг «AI-лаборатории».
- **Фаза 2 — Auth + онбординг**: Supabase Auth (email magic-link + Google),
  защищённые маршруты, страница /dashboard со списком проектов, деплой на Vercel.
- **Фаза 3 — Workspace (экран как на фото)**: file-tree, Monaco-редактор,
  вкладки Код/Архитектура/Логи/Аналитика, панель ИИ-чата, топбар с
  Запустить/Скачать ZIP/Поделиться. Пока со стаб-данными.
- **Фаза 4 — ИИ code-gen движок**: Claude генерирует/правит мульти-файловый
  проект бота (стриминг + «ход рассуждений»), запись файлов, авто-фикс по логам.
- **Фаза 5 — Запуск и логи**: изолированное выполнение бота (сначала контейнер/
  песочница-заглушка), стрим логов в UI, Скачать ZIP.
- **Фаза 6 — Stripe подписки**: тарифы Basic/Pro (monthly), Checkout, Customer
  Portal, webhooks → `subscriptions`, лимиты по плану (число проектов, ИИ-квоты).
- **Фаза 7 — Аналитика + мониторинг**: события PostHog, дашборд «Аналитика»,
  Sentry алерты, session replay.
- **Фаза 8 — Легал + запуск**: Privacy Policy, Terms of Service, cookie-consent
  (GDPR), SEO/OG, страницы ошибок, финальная полировка и нагрузочная проверка.

## Принципы качества (чтобы багов было минимум)
1. TypeScript strict + Zod на всех границах (API, формы, ответы ИИ, webhooks).
2. GitHub Actions блокирует мёрж при падении lint/typecheck/build.
3. Vercel Preview-деплой на каждый PR — визуальная проверка до прода.
4. Sentry ловит рантайм-ошибки; feature-флаги через PostHog.
5. Секреты только в env, RLS в Supabase, Stripe webhook-подпись проверяется.
6. Мелкие атомарные PR, каждая фаза — отдельный рабочий кусок.

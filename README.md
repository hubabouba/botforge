# BotConstruct.io — ИИ-конструктор Telegram/Discord ботов

Веб-платформа, где боты собираются **визуально из блоков**, а **ИИ (Claude)**
помогает генерировать сценарий из текстового описания. Одна JSON-схема
исполняется на **Telegram** (grammY) и **Discord** (discord.js) единым движком —
платформа сама запускает и держит ботов онлайн.

## Стек
- **Next.js 15** (App Router) + **TypeScript** — фронт и бэк на одном языке
- **React Flow** (`@xyflow/react`) — визуальный редактор схем
- **Prisma + SQLite** (dev) — данные ботов, схемы, логи *(легко переключить на Postgres)*
- **grammy** / **discord.js** — адаптеры платформ
- **@anthropic-ai/sdk** — генерация схем (`claude-sonnet-5`) и нода `ai_reply` (`claude-haiku-4-5`)

## Требования
- **Node.js 18.18+** (сейчас на машине не установлен — поставьте с https://nodejs.org)
- Токен Telegram-бота (от @BotFather) и/или Discord-бота (Developer Portal)
- Ключ Anthropic API — https://console.anthropic.com

## Быстрый старт
```powershell
# 1. Скопировать переменные окружения и заполнить ключи
Copy-Item .env.example .env
#   ANTHROPIC_API_KEY=...   AUTH_SECRET=<длинная случайная строка>

# 2. Установить зависимости
npm install

# 3. Создать БД (SQLite) по схеме Prisma
npm run db:push

# 4. Запустить дев-сервер
npm run dev
# → http://localhost:3000
```

## Как пользоваться
1. `/dashboard` → создать бота (Telegram или Discord).
2. В редакторе: перетаскивать блоки из левой панели **или** описать бота словами
   и нажать **«✨ Собрать с ИИ»** — Claude сгенерирует схему.
3. Настроить блоки в правой панели; соединять их линиями (порядок выполнения).
4. Вставить **токен бота** → **«Сохранить токен»** (хранится зашифрованным, AES-256-GCM).
5. Нажать **«Запустить»**. Написать боту `/start` — он ответит по схеме.
   Логи видны внизу редактора. **«Остановить»** — выключить бота.

## Блоки (типы нод)
| Блок | Назначение |
|------|-----------|
| Триггер | старт сценария: команда `/start`, ключевое слово или любое сообщение |
| Сообщение | отправить текст (плейсхолдеры `{user.name}`, `{vars.x}`) |
| Кнопки | inline-кнопки / компоненты Discord, ведущие к другим блокам |
| Условие | ветвление по тексту сообщения или переменной |
| ИИ-ответ | ответ, сгенерированный Claude по системному промпту |
| Переменная | сохранить значение в сессию пользователя |

## Архитектура
```
src/
├─ app/
│  ├─ page.tsx                     лендинг
│  ├─ dashboard/                   список ботов
│  ├─ builder/[botId]/             визуальный редактор
│  └─ api/                         bots CRUD, token, deploy, ai/generate, logs
├─ components/builder/             Builder, палитра, панель свойств, нода
├─ lib/
│  ├─ schema/types.ts              Zod-схема графа — ЕДИНЫЙ источник истины
│  ├─ ai/claude.ts                 Claude: текст → граф, ai_reply
│  ├─ runtime/engine.ts            интерпретатор графа (платформо-независимый)
│  ├─ runtime/telegram.ts          адаптер grammY
│  ├─ runtime/discord.ts           адаптер discord.js
│  └─ runtime/runner.ts            менеджер запущенных ботов + сессии
├─ lib/crypto.ts                   AES-256-GCM для токенов
└─ db/prisma.ts                    Prisma client
```

## Замечания / дорожная карта
- **Хостинг ботов (MVP):** боты работают в процессе Next.js через long polling —
  удобно для локальной разработки. Для продакшена (24/7, много ботов) вынести
  `runner.ts` в отдельный worker-сервис + вебхуки + очередь. Абстракция
  `RunningBot` это позволяет без переписывания движка.
- **Discord:** включите *Message Content Intent* в Developer Portal.
